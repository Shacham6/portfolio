**Status:** Work in progress, developed in my free time.

A terminal-based roguelike where every entity — doors, enemies, computers — exposes a filesystem and accepts commands. The core challenge: building a complete game engine from scratch using only Go and Raylib, with no additional dependencies.

This project demonstrates systems-level programming and algorithmic problem-solving across several domains:

**Procedural Generation**

Dungeon layouts and item placements are generated algorithmically, ensuring each playthrough presents unique spatial challenges.

**Terminal Interface**

Implemented a fully functional in-game terminal that executes commands against a virtual filesystem. Every interactive entity exposes its state through this interface.

**Network Simulation**

Entities can establish connections to other entities, simulating network access. The terminal serves as the primary interface for remote interaction.

**Command-Driven Mechanics**

Game state changes through terminal commands — unlocking doors, disabling security systems, manipulating entity behavior. The command system integrates directly with game mechanics.

  "/portfolio/projects/rogue-hacking-game/game_hack_everything.png"

**Virtual Filesystem**

Each entity maintains its own filesystem with configuration files, passwords, and state data. Players interact with these systems to progress, creating emergent gameplay through file manipulation.

  "/portfolio/projects/rogue-hacking-game/game_filesystem.GIF",

**Technical Implementation**

Beyond game mechanics, this project required implementing several core systems from scratch:

- **Immediate-mode GUI framework** — real-time interface rendering without retained-mode overhead
- **Shadowcast algorithm** — field-of-view calculation for entity vision and line-of-sight
- **A\* (A-star) pathfinding** — entity navigation through procedurally generated spaces
- **Entity Behavior Tree (Game AI)** - turning the entities into proper characters, with routines and exceptional behaviors
